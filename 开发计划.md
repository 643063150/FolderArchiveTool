# FolderArchiveTool 开发计划

> **项目名称**：FolderArchiveTool — 月度文件压缩归档工具  
> **平台**：Windows  
> **技术栈**：Python + PySide6  
> **设计风格**：Material Design  
> **文档版本**：v1.0  
> **日期**：2026-07-20

---

## 一、项目概述

### 1.1 背景
在 Windows 服务器上，存在以日期格式命名的文件（如 `2026-07-20.log`、`2026-07-20-data.csv`），这些文件体积较大，需要按月打包压缩为 zip 包并安全删除原文件，同时发送邮件通知管理员。

### 1.2 核心目标
- 自动扫描指定目录下的日期格式文件
- 按年月分组打包为 zip 压缩包
- **压缩后严格校验**，确认无误后才删除原文件
- 支持多邮件服务器（第三方 + 自建）同时/备用发送通知
- 可视化界面，支持 Material Design 设计语言
- 支持开机自启
- **每个功能均支持手动执行 + 定时自动执行双模式**

### 4.1 文件扫描模块（file_scanner.py）

**职责**：扫描目标目录，识别日期格式文件，按年月分组

```python
class FileScanner:
    """文件扫描器：识别日期格式文件并按月分组"""
    
    # 支持的日期正则模式
    DATE_PATTERNS = [
        r'(\d{4})-(\d{2})-(\d{2})',     # 2026-07-20
        r'(\d{4})_(\d{2})_(\d{2})',     # 2026_07_20
        r'(\d{4})(\d{2})(\d{2})',        # 20260720
        r'(\d{4})\.(\d{2})\.(\d{2})',   # 2026.07.20
    ]
    
    def scan(self, directory: str, recursive: bool = False) -> dict:
        """
        扫描目录，返回按月分组的文件列表
        return: {
            '2026-07': [file_info, ...],
            '2026-06': [file_info, ...],
        }
        """
    
    def scan_specific_month(self, directory: str, year: int, month: int) -> list:
        """扫描指定年月的文件"""
    
    def get_file_info(self, filepath: str) -> dict:
        """获取文件信息：路径、大小、修改时间、日期键"""
```

**手动执行入口**：UI 页面提供「扫描文件」按钮，点击后显示扫描结果预览

### 4.3 校验模块（validator.py）

**职责**：压缩后双重校验，确保文件完整性

```python
class Validator:
    """压缩包校验器"""
    
    def validate_archive(self, archive_path: str, original_crc_map: dict,
                         original_file_list: list) -> dict:
        """
        双重校验：
        1. 文件数量/名称比对
        2. CRC32 值逐文件比对
        return: { 'valid': bool, 'errors': list, 'details': dict }
        """
    
    def verify_zip_integrity(self, archive_path: str) -> bool:
        """测试 zip 包是否损坏"""
    
    def extract_crc_map(self, archive_path: str) -> dict:
        """从 zip 中提取所有文件的 CRC32"""
```

**安全删除原则**：
- 校验通过 → 删除原文件
- 校验失败 → **绝不删除**，记录错误日志，发送告警邮件
- 每次校验结果写入日志备查

```
    │
    ▼
[1] 扫描源目录 → 获取按月分组文件列表
    │
    ▼
[2] 过滤已归档月份（检查输出目录是否已存在对应zip）
    │
    ▼
[3] 压缩打包 → 显示进度 → 生成 CRC 映射表
    │
    ▼
[4] 校验压缩包 → CRC 比对 + 文件数量比对
    │
    ├── 校验通过 ──→ [5] 安全删除原文件
    │                      │
    │                      ▼
    │                 [6] 发送成功通知邮件
    │
    └── 校验失败 ──→ [6] 发送告警邮件（不删除原文件）
```

### 6.2 手动流程（用户点击）

```
用户点击「立即执行」
    │
    ▼
弹出确认对话框（显示将要归档的月份和文件数量）
    │
    ▼
用户确认 → 执行上述 [1]-[6] 流程
    │
    ▼
完成后弹出结果摘要 + Toast 通知
```

## 十、风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 超大文件压缩内存溢出 | 程序崩溃 | 使用流式压缩，分块读写 |
| 压缩过程中断（断电/崩溃） | 文件损坏 | 先写临时文件，完成后重命名 |
| 邮件服务器全部失败 | 通知不到 | 本地日志兜底 + 重试机制 |
| 误删文件 | 数据丢失 | 回收站模式 + 备份模式可选 |
| 多服务器时钟不同步 | 定时不准 | 使用 NTP 校准或本地时间 |
| PyInstaller 被杀毒误报 | 无法运行 | 代码签名 + 白名单申请 |

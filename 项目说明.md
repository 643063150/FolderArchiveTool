## 二、核心层（core/）

### 2.1 config_manager.py — 配置管理

| 功能 | 说明 |
|---|---|
| 单例模式 | 全局唯一实例，确保配置一致性 |
| 加载/保存 | 读写 `config.json`，不存在则从默认模板创建 |
| 密码加密 | 使用 Fernet 对称加密存储邮件密码 |
| 路径访问 | 支持点号路径（如 `mail.servers.0.host`） |
| 导入/导出 | 配置可导出为 JSON 文件，支持导入恢复 |

**主要方法：**
- `get(key_path, default)` — 获取配置项
- `set(key_path, value)` — 设置配置项
- `save()` — 保存到文件（线程安全）
- `import_config(path)` / `export_config(path)` — 导入/导出

---

### 2.2 logger_setup.py — 日志系统

| 功能 | 说明 |
|---|---|
| 文件日志 | RotatingFileHandler，自动轮转（10MB/文件，保留5个） |
| 控制台日志 | StreamHandler，输出到 stdout |
| UI 日志 | 通过 `pyqtSignal` 线程安全投递到主线程 |
| 日志级别 | INFO / WARNING / ERROR / DEBUG |

**线程安全机制：**
- 使用 `LogEmitter(QObject)` + `log_signal = Signal(str, str)`
- 后台线程发射信号 → Qt 自动排队到主线程 → 更新 UI

---

### 2.3 file_scanner.py — 文件扫描

| 功能 | 说明 |
|---|---|
| 日期识别 | 支持多种格式：`2026-07-20`、`2026_07_20`、`20260720`、`2026.07.20` |
| 按月分组 | 按 `YYYY-MM` 分组文件 |
| 递归扫描 | 可选递归子目录 |
| 进度回调 | `progress_callback(current, total, filepath)` |

**主要方法：**
- `scan(directory, pattern, recursive)` → `{month: [file_info, ...]}`
- `scan_specific_month(directory, year, month)` → 指定月份文件

---

### 2.4 archiver.py — 压缩引擎

| 功能 | 说明 |
|---|---|
| ZIP 压缩 | 使用标准库 `zipfile`，支持压缩级别 0-9 |
| 7z 压缩 | 可选内嵌 `7zr` 二进制（更高压缩比） |
| 临时文件 | 先写 `.tmp` 文件，完成后原子重命名（防止中断损坏） |
| CRC32 映射 | 压缩时计算每个文件的 CRC32，用于后续校验 |

**主要方法：**
- `create_archive(file_list, output_path, archive_name, level)` → 压缩结果字典

---

### 2.5 validator.py — CRC32 校验

| 功能 | 说明 |
|---|---|
| 完整性检查 | `testzip()` 检测压缩包是否损坏 |
| 双重校验 | 1. 文件数量/名称比对 → 2. CRC32 逐文件比对 |
| CRC 提取 | 从 zip 中提取每个文件的 CRC32 值 |

**主要方法：**
- `validate_archive(archive_path, original_crc_map, original_file_list)` → 校验结果
- `verify_zip_integrity(archive_path)` → 是否损坏

---

### 2.6 file_cleaner.py — 安全删除

| 功能 | 说明 |
|---|---|
| 校验前置 | 仅删除校验通过的文件 |
| 回收站模式 | 使用 Windows API 移入回收站（可恢复） |
| 备份模式 | 先复制到备份目录，再删除原文件 |
| 清理旧备份 | 自动清理超过保留天数的备份 |

**主要方法：**
- `safe_delete(file_list, validator_result)` → 删除结果
- `safe_delete_with_backup(file_list, backup_dir, validator_result)` → 带备份删除

---

### 2.7 mail_sender.py — 邮件发送

| 功能 | 说明 |
|---|---|
| 多服务器 | 支持同时发送（全部服务器）或主备切换 |
| SSL/TLS | 支持 SSL 和 STARTTLS |
| 发件人名称 | 自定义发件人显示名（如"服务器A-归档系统"） |
| HTML 报告 | 构建归档报告 HTML 邮件正文 |
| 详细日志 | 连接、登录、发送全过程日志 |

**主要方法：**
- `send_notification(subject, body, attachment)` → 发送结果
- `test_connection(server_config)` → 测试服务器连接
- `build_archive_report(archive_result, clean_result, validator_result)` → HTML 报告

---

### 2.8 scheduler.py — 定时调度

| 功能 | 说明 |
|---|---|
| APScheduler | 基于 `BackgroundScheduler`，支持 cron 表达式 |
| 任务类型 | 每月 / 每周 / 每天 / 自定义 Cron |
| 持久化 | 任务保存到 `config.json`，重启后自动恢复 |
| 失败重试 | `coalesce=True` 合并错过的触发 |
| 单实例 | `max_instances=1` 防止并发执行 |

**主要方法：**
- `add_monthly_job(day, hour, minute)` → 添加每月任务
- `add_weekly_job(day_of_week, hour, minute)` → 添加每周任务
- `add_custom_cron(cron_expr)` → 添加自定义任务
- `load_jobs()` → 从配置恢复任务
- `list_jobs()` → 列出所有任务
- `trigger_now()` → 立即执行一次

---

### 2.9 auto_start.py — 开机自启

| 功能 | 说明 |
|---|---|
| 注册表方式 | 写入 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` |
| 任务计划程序 | 创建 Windows 计划任务（自动提权，无 UAC 弹窗） |
| 降级策略 | 任务计划失败时自动降级为注册表方式 |

**主要方法：**
- `enable_task_scheduler()` → 启用任务计划（弹 UAC 一次）
- `disable_task_scheduler()` → 禁用
- `enable_registry()` → 启用注册表方式
- `is_task_scheduler_enabled()` → 检查是否启用

---

### 2.10 archive_service.py — 流程编排

| 功能 | 说明 |
|---|---|
| 完整流程 | 扫描 → 压缩 → 校验 → 删除 → 邮件通知 |
| 当月跳过 | 自动跳过当月（数据可能不完整） |
| 模式标记 | `[手动]` / `[自动]` 前缀区分执行模式 |
| 进度回调 | `progress_callback(stage, current, total, message)` |
| 并发保护 | `_is_running` 标志防止重复执行 |

**主要方法：**
- `scan_files()` → 扫描文件
- `run_full_archive(target_month, mode)` → 执行完整归档流程

### 2.11 windows_service.py — Windows 服务

| 功能 | 说明 |
|---|---|
| sc.exe 注册 | 使用 Windows 内置 `sc.exe` 创建服务 |
| 自动重启 | 配置服务失败自动重启（60秒间隔，重试3次） |

## 六、安全机制

> **校验不通过 → 绝不删除原文件**

1. **CRC32 双重校验**
   - 文件数量/名称比对
   - CRC32 逐文件比对

2. **压缩包完整性**
   - 先写 `.tmp` 文件
   - 完成后原子重命名

3. **邮件密码加密**
   - Fernet 对称加密
   - 密钥存储在本地 `config.key`
